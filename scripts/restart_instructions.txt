L'erreur 500 sur /api/user/me etait due a une incompatibilite de types chrono:
- Base de donnees: TIMESTAMPTZ (timestamp with timezone)
- Code Rust: NaiveDateTime (timestamp sans timezone)
- SOLUTION: Changement vers DateTime<Utc> dans user_model.rs

CORRECTION APPLIQUEE:
- Fichier backend/src/models/user_model.rs modifie
- NaiveDateTime -> DateTime<Utc> pour created_at et updated_at
- Compilation reussie avec cargo check

Le serveur backend doit etre redemarre pour appliquer les changements!

ETAPES DE REDEMARRAGE:

4. Relancez le serveur: cargo run
